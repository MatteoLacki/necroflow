necroflow feature tree
======================

necroflow
├── Core model
│   ├── NodeType typed file/artifact classes
│   ├── NodeType subtype families for format contracts
│   ├── NodeType union inputs for alternative type contracts
│   ├── Node objects with parents, config, command, output path, state
│   ├── Module-level rule declarations
│   │   ├── shell command API: @command(...)
│   │   ├── built-in decorators: @text_file / @symlink_file
│   │   ├── explicit built-in factories: text_file_rule / symlink_file_rule
│   │   ├── lint-clean name = output(NodeType) declarations
│   │   ├── typed Inputs / Outputs
│   │   ├── multi-output rules
│   │   ├── co-output execution once per rule call
│   │   ├── Constraints: threads, ram, custom resources
│   │   ├── repeat=N compatibility metadata
│   │   └── built-in text_file rules for generated config files
│   ├── Pipeline
│   │   ├── attribute-based node registration
│   │   ├── pipeline labels for manifests and requests
│   │   ├── author-declared presentation sections via P.section(...)
│   │   └── conditional Python pipeline construction
│   └── DAG
│       ├── multi-pipeline aggregation
│       ├── shared-node deduplication
│       ├── requested target nodes
│       └── ASCII graph rendering / save
│
├── Commands
│   ├── input/output path placeholders
│   ├── config placeholders
│   ├── {workdir} retained side-output directory
│   ├── constraint placeholders
│   │   ├── {threads}, default 1
│   │   ├── {ram} and other declared constraints
│   │   └── {constraint:name} forced constraint lookup
│   ├── escaped literal shell braces via {{...}}
│   ├── static shell templates
│   ├── Python callbacks receiving resolved CommandArgs
│   └── configurable shellpath
│
├── Caching and paths
│   ├── content-addressed node paths
│   ├── full framed fingerprint-v2 digests with 16-char path form
│   ├── filesystem cache as source of truth
│   ├── parent/content/config/type-aware fingerprints
│   ├── optional project-wide .fingerprint function
│   ├── explicit shellpath included in command fingerprints
│   ├── path length validation
│   ├── output content hashes
│   ├── invalidator tokens on NodeType
│   └── .rip metadata directories
│
├── Execution engine
│   ├── parallel execution
│   ├── resource caps

│   ├── scheduler protocol

│   ├── connected-component scheduler
│   ├── FIFO scheduler
│   ├── custom schedulers
│   │   └── Python callables can implement resource-aware ordering policies
│   ├── dry-run mode

│   ├── missing-output detection
│   ├── job logs
│   ├── execution reports
│   │   ├── .rip/run.toml node-local runtime stats
│   │   ├── results/<job>/execution.toml per-run summaries
│   │   └── duration and output-size accounting
│   ├── output/provenance graph writing
